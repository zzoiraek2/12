# FOBL MATCH 허수거래 방지 및 거래온도 정책

> 본 문서는 FOBL MATCH 서비스의 허수거래, 반복 취소, 미응답, 미입금, 허위증빙을 방지하기 위한 운영정책과 고객별 거래온도 산정 기준을 정리한 내부 운영 기준입니다.

---

## 1. 목적

FOBL MATCH 서비스는 USDT 매수자와 USDT 매도자의 거래 의사를 매칭하고, USDT 전송, 수수료 입금, 원화 보내기, 이체확인증 등록, 원화 입금 확인, 니모닉 열람 절차를 관리한다.

허수거래 방지 정책은 다음 목적을 가진다.

1. 실제 거래 의사가 없는 신청과 반복 취소를 줄인다.
2. 매칭 후 미응답, 미입금, 지연 이행을 제한한다.
3. 정상 거래자의 매칭 품질과 거래 안정성을 높인다.
4. 허위증빙과 반복 미이행 회원을 조기에 식별한다.
5. 고객에게는 단순한 거래온도만 노출하고, 상세 산출근거는 어드민에서 감사 가능하게 관리한다.

## 3. 고객 노출 원칙

고객 화면에는 거래온도만 노출한다.

```text
거래온도 53.7℃
```

고객 안내 문구는 다음 수준으로 제한한다.

```text
거래온도는 거래 완료, 취소, 미응답, 분쟁, 운영정책 위반 등을 종합해 산정되는 신뢰 지표입니다.
```

고객에게 직접 노출하지 않는 항목은 다음과 같다.

| 항목 | 비노출 사유 |
|---|---|
| 완료율 | 개별 거래 패턴 추정 가능 |
| 취소율 | 과도한 낙인 또는 분쟁 가능 |
| 미응답/미입금률 | 내부 운영 판단 기준 |
| 분쟁률 | 명예훼손, 낙인, 민원 가능 |
| 제재 이력 | 운영/준법 검토 정보 |
| 산출 상세 점수 | 정책 악용 가능 |

## 5. 거래온도 산식

```text
거래온도 =
clamp(
  36.5
  + 거래건수보정 × (
      완료점수
    + 정시이행점수
    + 긍정평가점수
    - 취소패널티
    - 미응답/미입금패널티
    - 분쟁패널티
    - 제재패널티
  ),
  0,
  99.9
)
```

### 5.1 거래건수 보정

```text
거래건수보정 = min(1, sqrt(최근 평가대상 거래건수 / 100))
```

| 거래건수 | 반영 비율 |
|---:|---:|
| 10건 | 약 31.6% |
| 25건 | 50.0% |
| 100건 이상 | 100.0% |

거래건수 보정은 신규 회원의 1~2건 거래만으로 온도가 과하게 상승하거나 하락하는 것을 방지하기 위한 장치다.

### 5.2 점수 구성

| 항목 | 산식 예시 | 설명 |
|---|---:|---|
| 완료점수 | `34 × (보정완료율 - 0.5)` | 거래를 끝까지 완료했는지 |
| 정시이행점수 | `16 × (보정정시이행률 - 0.5)` | USDT, 수수료, 원화, 확인 버튼 등 제한시간 준수 |
| 긍정평가점수 | `12 × (보정긍정평가율 - 0.5)` | 상대방 평가, 운영자 정상평가 |
| 취소패널티 | `26 × 보정취소율` | 매칭 전후 취소 |
| 미응답/미입금패널티 | `34 × 보정미이행률` | 미응답, USDT 미입금, 수수료 미입금, 원화 미이체 |
| 분쟁패널티 | `42 × 보정분쟁률` | 분쟁 전환, 귀책 분쟁 |
| 제재패널티 | 제재건별 고정 차감 | 허위증빙, 반복 미이행, 운영자 제재 |

### 5.3 보정 비율

```text
보정완료율 = (완료건수 + 5) / (평가대상건수 + 10)
보정정시이행률 = (정시이행건수 + 5) / (평가대상건수 + 10)
보정긍정평가율 = (긍정평가건수 + 5) / (평가대상건수 + 10)

보정취소율 = (취소건수 + 1) / (평가대상건수 + 20)
보정미이행률 = (미응답·미입금건수 + 0.4) / (평가대상건수 + 20)
보정분쟁률 = (분쟁건수 + 0.5) / (평가대상건수 + 50)
```

보정 비율은 거래건수가 적은 회원의 비율이 과장되는 것을 막기 위해 사용한다.

## 11. 운영 유의사항

1. 거래온도는 고객에게 신뢰 지표로만 제공하며, 세부 산출근거는 고객에게 공개하지 않는다.
2. 온도 산식과 패널티 기준은 정책 버전으로 관리한다.
3. 정책 변경 시 변경 전후 산식, 적용 시작일시, 승인자를 기록한다.
4. 온도 하락에 따른 제한은 자동 제한과 운영자 검토 제한을 구분한다.
5. 허위증빙, 중대한 제재, AML/FDS 제한은 거래온도와 무관하게 즉시 제한할 수 있다.
6. 고객 민원 발생 시 어드민 산출 스냅샷과 이벤트 로그를 기준으로 설명 가능해야 한다.
